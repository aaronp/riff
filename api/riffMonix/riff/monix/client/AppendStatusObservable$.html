    Create a feed of 'AppendStatus' based off the inputs coming into a node, the node's log results, and an initial LogAppendResult

    Create a feed of 'AppendStatus' based off the inputs coming into a node, the node's log results, and an initial LogAppendResult

    clusterSize

    used to populate the first AppendStatus event

    nodeInput

    the data coming into the node, which will be filtered on AppendEntriesResponse messages

    maxCapacity

    the buffer size of append responses to keep. In practice this should be < 10 elements, so 'maxCapacity' is defaulted quite low. If a client appends a LOT of data though, there could be a lot of ACKS and commits, so this should be driven by a function of the size of the cluster, batch size, and number of elements appended

    returns

    and observer which should be given a single LogAppendResult from having tried an append entry, as well as an Observable of AppendStatus responses
